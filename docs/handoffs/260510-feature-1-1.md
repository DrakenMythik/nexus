# 🚀 NEXUS PROJECT HANDOFF & CONTEXT

## 📌 Project Overview
**Name:** Nexus (Full-Stack Workout & Health Tracker)
**Methodology:** AI-Assisted Development using Cursor (`agent:architect` -> `agent:coder` pipeline).
**Architecture:** Strict Feature-Sliced Design (FSD). Dependencies flow strictly downward (app -> pages -> widgets -> features -> entities -> shared). FSD boundaries are enforced via `eslint-plugin-boundaries`.

## 🛠️ Tech Stack
* **Frontend:** React 19, Vite, TypeScript, Tailwind CSS, React Router v7.
* **State & Data Fetching:** Zustand (client state/session), React Query (server cache/profiles).
* **Backend & Auth:** Supabase (PostgreSQL, Auth, RLS).
* **Testing:** Vitest + Testing Library (Unit), Cypress (E2E).

## ✅ Current State: What is DONE
We have just completed **Feature 1.1: Secure Authentication Setup**. 
The repository currently contains:

**1. Backend / Database (Local Supabase Container):**
* Migrations applied for `public.profiles`, `public.workout_sessions`, and `public.health_metrics`.
* Strict Row-Level Security (RLS) is active on all tables (`auth.uid() = user_id`).
* Schema visibility patched (`REVOKE SELECT ON ... FROM anon;`).
* Trigger `handle_new_user()` syncs new auth users to `public.profiles`.

**2. Frontend Infrastructure (FSD Plumbing):**
* `src/shared/`: Environment validation (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) and pure Supabase browser client creation.
* `src/entities/user/`: Types, Zustand `useUserStore` (session, userId, profile), and React Query fetching logic.
* `src/app/`: `AppProviders` (React Query + Supabase context), `AuthStateBridge` (syncs Supabase auth state to Zustand), and `App.tsx` routing.

**3. UI & Routing (Pages & Features):**
* `src/features/`: Four isolated slices (`auth-by-email`, `auth-by-google`, `auth-by-apple`, `auth-by-microsoft`).
* `src/pages/`: `LoginPage`, `RegisterPage`, and `AuthCallbackPage` (handles OAuth PKCE flow).
* All routing utilizes `react-router-dom` `<Link>` components for SPA performance.

**4. Testing & DevOps:**
* `.env*` is properly gitignored.
* Vitest is configured with `jsdom`; tests exist for the Zustand store.
* Cypress is configured; a baseline E2E test (`auth.cy.ts`) exists for a failed login flow to verify UI feedback.
